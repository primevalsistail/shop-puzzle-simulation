# 要件定義書 — Cycle 2: UI/UX 全面リニューアル

## 意図分析サマリー

| 項目 | 内容 |
|------|------|
| **ユーザーリクエスト** | UI を全画面化・新レイアウトに変更。キャラエリア・メッセージウィンドウ・カテゴリフィルタを追加 |

## 機能要件

### FR-UI-01: ウィンドウ全体化
- **FR-UI-01-1**: Phaser の `Scale.FIT` モードを使用し、16:9 アスペクト比を維持しながらブラウザウィンドウ全体に拡張
- **FR-UI-01-2**: `autoCenter: CENTER_BOTH` で常に中央配置
- **FR-UI-01-3**: 内部解像度は 1280×720 基準のまま（全座標系はこの解像度で計算）
- **FR-UI-01-4**: ウィンドウリサイズ時に自動追従

### FR-UI-02: 新レイアウト

#### 左パネル（商品リスト + フィルタ）
- **FR-UI-02-1**: 画面左端から幅 220px
- **FR-UI-02-2**: カテゴリフィルタ: 食品・工芸品・雑貨などのジャンル別トグルボタン（複数選択可）
- **FR-UI-02-3**: フィルタ状態に応じてアイテムリストをリアルタイム絞り込み
- **FR-UI-02-4**: 「全表示」リセットボタン

#### 中央エリア（商品棚グリッド）
- **FR-UI-02-5**: 左パネル右端〜キャラストリップ左端の範囲を最大限使用
- **FR-UI-02-6**: グリッドのセルサイズを利用可能な中央幅・高さに合わせて動的計算
- **FR-UI-02-7**: グリッドは中央エリア内で中央揃え配置

#### キャラストリップ（中央右の縦ストリップ）
- **FR-UI-02-8**: 右パネル左側に隣接する縦長エリア（幅 ~110px）
- **FR-UI-02-9**: 上半分: 店番主人公エリア（レジ前に座っているキャラのプレースホルダー）
- **FR-UI-02-10**: 下半分: 来店客エリア（歩いて来店するキャラのプレースホルダー）
- **FR-UI-02-11**: 現時点はプレースホルダー（アイコン+ラベルテキスト）。将来のアニメーション実装に向けた API 設計

#### 右パネル
- **FR-UI-02-12**: 画面右端から幅 200px
- **FR-UI-02-13**: 上段: 時間・売上表示（HUD）
- **FR-UI-02-14**: 中段: キャラ絵エリア（プレースホルダー画像領域）
- **FR-UI-02-15**: 下段: ボタン群（仕入れ・クラフト・進める）＋アイコンボタン

#### メッセージウィンドウ（下部全幅）
- **FR-UI-02-16**: 画面下部、全幅、高さ ~110px
- **FR-UI-02-17**: 販売ログを表示（「りんごが 1 個売れた！ +¥100」）
- **FR-UI-02-18**: 将来のイベントメッセージ表示にも対応できる拡張可能な設計
- **FR-UI-02-19**: 最新メッセージ n 件（例: 5件）をスタック表示、古いものから消える

### FR-UI-03: 商品リスト カテゴリフィルタ

アイテムデータ（`data/items.ts`）のカテゴリ情報を活用:
- **FR-UI-03-1**: アイテムに `category` プロパティ（例: `'food' | 'craft' | 'goods'`）を追加
- **FR-UI-03-2**: InventoryPanel にカテゴリトグルボタンを追加
- **FR-UI-03-3**: 複数カテゴリを同時選択可（OR フィルタ）
- **FR-UI-03-4**: 全カテゴリ選択時 = 全表示

### FR-UI-04: キャラストリップ（将来拡張向け）
- **FR-UI-04-1**: `CharacterStrip` コンポーネントを新規作成
- **FR-UI-04-2**: `setShopkeeper(placeholder | animated)` のような API を持つ
- **FR-UI-04-3**: `addCustomer(customerId)` / `removeCustomer(customerId)` で客の出現/退場を制御できる設計
- **FR-UI-04-4**: 初回実装はプレースホルダー（四角 + アイコン + テキスト）

### FR-UI-05: メッセージログ
- **FR-UI-05-1**: `MessageLog` コンポーネントを新規作成
- **FR-UI-05-2**: `addMessage(text, type?)` API（type: `'sale' | 'event' | 'info'`）
- **FR-UI-05-3**: 色分け: sale=黄色、event=シアン、info=グレー
- **FR-UI-05-4**: 販売イベント (`FLOOR_SLOT_SOLD`) を自動購読してメッセージ追加
- **FR-UI-05-5**: 最新 5 件を表示、それ以前はフェードアウト

## 非機能要件

### NFR-UI-01: 既存ゲームロジック無変更
- ゲームロジック（FloorGrid, PlacementManager, CustomerSimulator 等）は一切変更しない
- テスト 101 件の継続パスを維持

### NFR-UI-02: 段階的移行
- 既存 UI コンポーネント（HUD, InventoryPanel 等）の座標を新レイアウトに合わせて更新（再実装ではなく移設）

---

## スコープ外（今回含まない）

- キャラクター実際のアニメーション・グラフィック（プレースホルダーのみ）
- イベントシステム本体（メッセージウィンドウの受け口だけ用意）
- フロアエリア拡張（13×10 グリッド対応）
- セルサイズのリアルタイムリサイズ（Scale.FIT の恩恵でブラウザ側でスケール）
